# 视频标题标签提取技术方案（设计文档）

## 1. 背景与目标

### 1.1 业务目标
从视频标题中自动提取标签，满足两类能力：

1. 统计高频词：识别“经常出现”的关键词。
2. 语义归并：把“词义相近”的词聚合到同一标签。

### 1.2 输出目标

1. 为每个视频生成一组 `AI 标注标签`（可人工校正）。
2. 形成“标签词典”与“别名关系”。
3. 可持续增量更新，不破坏当前 SQLite 架构和手工标签逻辑。

## 2. 范围与非目标

### 2.1 范围（本方案）

1. 输入仅使用标题文本：`title_guess + filename`。
2. 采用本地小模型（离线）进行语义相似度计算。
3. 结果写入现有 `tags/video_tags` 体系，`source='ai_title'`。

### 2.2 非目标（后续阶段）

1. 暂不使用音频 ASR、帧 OCR、图像内容识别。
2. 暂不做跨语种复杂对齐（先聚焦中文+常见英文词）。
3. 暂不自动“覆盖”手工标签。

## 3. 当前系统约束

1. 存储：SQLite（`<库根>/.mm/library.db`）。
2. 已有表：`videos/tags/video_tags`，支持 `source` 字段区分标签来源。
3. 已有扫描流程与日志模块，可复用进度与可观测性能力。

## 4. 总体架构

```
标题数据读取 -> 文本清洗 -> 候选词抽取 -> 词频统计
                               -> 向量编码 -> 相似词聚类 -> 标签标准化
                               -> 视频标签回填 -> 持久化与日志
```

建议新增模块（后续编码阶段）：

1. `app/application/tag_mining_service.py`
2. `app/infrastructure/text_normalizer.py`
3. `app/infrastructure/tokenizer.py`
4. `app/infrastructure/embedding_model.py`
5. `app/infrastructure/tag_mining_repository.py`

## 5. 模型选型建议

## 5.1 推荐模型（本地）

1. `BAAI/bge-small-zh-v1.5`（优先）
2. 备选：`moka-ai/m3e-small`

理由：

1. 体积和效果平衡，CPU 可跑。
2. 适合短文本（词/短短语）语义相似计算。
3. 无需联网推理，便于桌面应用离线部署。

## 5.2 推理策略

1. 仅对候选词做 embedding（数量远小于视频总数）。
2. 启用本地缓存（词 -> 向量）减少重复计算。

## 6. 算法流程设计

## 6.1 数据采集

来源：

1. `videos.title_guess`
2. `videos.filename`（去扩展名）

构造 `effective_title`：

1. 优先 `title_guess`，若为空回退 `filename`。
2. 保留原文用于回溯。

## 6.2 文本标准化

顺序建议：

1. Unicode 归一化：`NFKC`
2. 去控制字符、重复空白
3. 全角/半角统一
4. 繁简统一（建议接入 `OpenCC`）
5. 去噪模式（可配置）：
   - 分辨率、码率、字幕源等技术噪声（如 `1080p`、`中字`）
   - 方括号/花括号装饰段
   - 明显站点前后缀

## 6.3 候选词抽取

采用“分词 + n-gram”混合策略：

1. 中文分词（`jieba` 或 `pkuseg`）。
2. 英文/数字词按正则切分并归一。
3. 补充 2~4 字中文 n-gram（用于分词漏召回）。

过滤规则（默认）：

1. 中文词长度 `< 2` 丢弃。
2. 停用词丢弃。
3. 全数字词（纯数字）丢弃。
4. 低频词先保留到统计阶段再裁剪。

## 6.4 词频统计

统计两个指标：

1. `TF`：词总出现次数。
2. `DF`：词覆盖视频数（更稳定）。

建议默认阈值：

1. `min_df = 2`（至少出现在 2 个视频）。
2. `top_k` 可按库规模动态设定。

## 6.5 语义归并（相近词）

步骤：

1. 对候选词编码为向量。
2. 计算余弦相似度。
3. 用并查集/图连通分量做聚类（`sim >= threshold`）。

默认阈值建议：

1. `0.80` 起步（偏保守）。
2. 可按人工验收结果调到 `0.78~0.85`。

## 6.6 标准标签选取

每个簇选择“主标签”：

评分函数建议：

```
score = 0.7 * normalized_df + 0.3 * normalized_tf
```

主标签规则：

1. 分数最高者为 canonical tag。
2. 其余词记录为 alias。

## 6.7 视频标签回填

对每个视频：

1. 标题词命中 alias/canonical 即映射到 canonical 标签。
2. 写入 `video_tags(source='ai_title')`。
3. 置信度可按命中类型设置：
   - 直接命中 canonical：`0.95`
   - alias 命中：`0.80~0.90`

注意：

1. 不删除 `source='manual'`。
2. `ai_title` 标签支持重复运行覆盖更新（同 source upsert）。

## 7. 数据库设计建议

## 7.1 复用现有表（必须）

1. `tags`
2. `video_tags`（利用 `source` 区分）

## 7.2 增量扩展表（建议）

1. `ai_tag_runs`
   - `id`
   - `started_at_epoch`
   - `finished_at_epoch`
   - `status`
   - `config_json`
   - `summary_json`
2. `tag_aliases`
   - `alias`
   - `tag_id`
   - `similarity`
   - `run_id`
3. `term_stats`
   - `term`
   - `tf`
   - `df`
   - `run_id`

作用：

1. 结果可审计、可回放、可对比不同阈值效果。

## 8. UI/交互设计建议

新增“标题标签提取”入口（建议在主工具栏）：

1. 开始提取按钮。
2. 参数面板（最小集）：
   - `min_df`
   - 相似度阈值
3. 进度显示：
   - 当前阶段（清洗/统计/聚类/写库）
   - 当前进度
4. 结果预览：
   - 标签簇（主标签 + 别名）
   - 可人工确认/拒绝（建议半自动）

## 9. 性能与资源预估

按 5,000~20,000 视频标题：

1. 文本清洗与分词：秒级到十几秒。
2. 向量编码：取决于候选词量（通常数百到几千）。
3. 相似度计算：词量较小时可全量；词量大时建议 ANN 或分批。

优化策略：

1. embedding 缓存（词哈希）。
2. 增量运行：只处理新增/变化标题。
3. 后台线程执行，避免阻塞 UI。

## 10. 质量评估方案

关键指标：

1. `Tag Precision@K`：人工抽样看标签是否准确。
2. `Coverage`：至少命中 1 个标签的视频占比。
3. `Cluster Purity`：同簇语义一致性。

上线策略：

1. 先灰度：只写入候选，不自动展示到主筛选。
2. 人工确认后再启用到默认筛选。

## 11. 风险与对策

1. 标题噪声大导致误标签  
对策：加强去噪词典 + 提高阈值 + 人工确认

2. 同形异义词误合并  
对策：保守阈值、按上下文二次判定、支持手工拆分簇

3. 模型资源占用高  
对策：小模型 + 缓存 + 增量处理

## 12. 分阶段实施计划

### Phase 1（可用版）

1. 清洗 + 分词 + 高频统计
2. 小模型相似词聚类
3. 写入 `tags/video_tags(source='ai_title')`

### Phase 2（可运营版）

1. 增加 `ai_tag_runs/tag_aliases/term_stats`
2. UI 增加参数和结果预览
3. 人工确认闭环

### Phase 3（增强版）

1. 引入 ASR/OCR/画面特征融合
2. 标签质量学习与自动阈值调优

## 默认配置（建议初值）

1. `min_df = 2`
2. `similarity_threshold = 0.80`
3. `max_tags_per_video = 8`
4. `source = 'ai_title'`

该配置偏保守，目标是“先减少误标，再逐步提高召回”。
